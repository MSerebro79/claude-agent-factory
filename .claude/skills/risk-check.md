# Risk Check

Процедура проверки рисков и failure modes для Reviewer.

## Контрольные вопросы

- Что произойдёт если LLM галлюцинирует?
- Что произойдёт если источник данных недоступен?
- Есть ли обработка ошибок и fallback-стратегия?
- Предусмотрен ли timeout для внешних вызовов?
- Как система ведёт себя при частичном отказе?
- Учтены ли риски из `libraries/risk_library.md`?

## Справочник по категориям рисков

Смотри `libraries/risk_library.md` по четырём категориям:

| Категория | Ключевые риски |
|---|---|
| **Данные** | Источник закрыт, данные устарели, объём недостаточен |
| **LLM** | Галлюцинации, нестабильный формат output, стоимость при масштабе |
| **Инфраструктура** | VPS упал, n8n завис, Supabase лимиты |
| **Продукт** | Боль не реальна, scope creep, MVP не используется |

## Шкала оценки

| Вероятность + Влияние | Рекомендация |
|---|---|
| Высокая + Высокое | Обязательная митигация — без неё REJECT или APPROVE WITH RISKS |
| Средняя + Высокое | Вынести явно в APPROVE WITH RISKS |
| Любая + Низкое | Фиксировать, не блокировать |

## Формат записи в review.md

В секции `### Failure Modes`:

```
| Статус | Комментарий |
|---|---|
| ✅ / ⚠️ / ❌ | [вопрос] — [ответ + митигация или отсутствие] |
```
