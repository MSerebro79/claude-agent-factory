# Risk Library

Типовые риски по категориям.
Reviewer использует как справочник при проверке blueprint.

---

## Данные

| Риск | Вероятность | Митигация |
|---|---|---|
| Источник закрыл API | Средняя | Иметь 2+ источника |
| Сайт заблокировал scraping | Высокая | Использовать официальные API |
| Данные устарели | Высокая | Добавить timestamp, TTL |
| Данных недостаточно для анализа | Средняя | Определить минимальный объём до старта |

## LLM

| Риск | Вероятность | Митигация |
|---|---|---|
| Галлюцинации | Высокая | Требовать ссылки на источник |
| Нестабильный формат output | Высокая | Structured output + валидация |
| Высокая стоимость при масштабе | Средняя | Кэширование, batching |
| Модель обновилась и поведение изменилось | Низкая | Пинить версию модели |

## Инфраструктура

| Риск | Вероятность | Митигация |
|---|---|---|
| VPS упал | Низкая | Мониторинг + алёрт в Telegram |
| n8n workflow завис | Средняя | Timeout + ручной перезапуск |
| Supabase лимиты бесплатного тира | Средняя | Следить за объёмом |

## Продукт

| Риск | Вероятность | Митигация |
|---|---|---|
| Боль пользователя не реальна | Высокая | Custdev до сборки |
| MVP не используется | Средняя | Запустить за 1 неделю, проверить |
| Scope creep | Высокая | Строгий MVP Scope в blueprint |
