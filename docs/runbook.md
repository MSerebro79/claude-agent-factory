# Runbook: Agent Factory

Короткая инструкция как пользоваться фабрикой.
Читай перед каждой новой сессией если забыла процесс.

---

## Быстрый старт

1. Открой папку `agent-factory/` в Claude Code
2. Claude Code прочитает `CLAUDE.md` автоматически
3. Напиши идею или задачу в свободной форме
4. Дальше следуй инструкциям агентов

---

## Три режима

**NEW PROJECT**
Новая идея. Designer задаёт максимум 5 вопросов → blueprint → Reviewer → ты одобряешь → Builder.

**EXISTING PROJECT**
Доработка существующего проекта из `/projects`. Designer читает существующие файлы → обновляет blueprint → Reviewer → ты одобряешь → Builder.

**PROJECT AUDIT**
Аудит без сборки. Только Reviewer. Выбери тип: Architecture / Product / Cost / Agent.

---

## Полный цикл шаг за шагом

```
1. Пишешь идею
2. Designer предлагает режим → ты подтверждаешь
3. Designer задаёт ≤5 вопросов (если Raw идея)
4. Designer создаёт blueprint.md
5. Reviewer создаёт review.md с вердиктом
6. Ты видишь оба файла вместе
7. Пишешь "одобряю" — Builder запускается
   или вносишь правки — цикл повторяется
   или REJECT + Override — Builder запускается несмотря на вердикт
8. Builder создаёт папку проекта
9. Definition of Done check
10. Открываешь новый проект в Claude Code
```

---

## После каждого прогона

Обнови `projects/index.md`:
- Добавь проект в таблицу
- Укажи статус и вердикт Reviewer
- Обнови Factory Stats

---

## Когда добавлять в libraries/

**components/** — если написал компонент который точно пригодится в следующем проекте (Telegram notifier, Supabase CRUD и т.д.)

**stack_decisions.md** — если принял нестандартное решение по стеку и хочешь помнить почему

**risk_library.md** — если столкнулся с новым риском которого не было в списке

**cost_library.md** — если появились новые данные по стоимости

---

## Pattern Review

После каждых 5 реализованных проектов:
1. Посмотри на `/projects/index.md` — какие паттерны повторяются?
2. Посмотри на `/libraries/components` — что переиспользовалось?
3. Обнови или добавь паттерны в `/patterns/`

---

## Если что-то пошло не так

**Blueprint получился плохой** → попроси Designer переделать конкретные секции

**Reviewer вынес REJECT но ты не согласна** → используй Human Override, Builder запустится

**Builder создал лишние файлы** → удали вручную, добавь заметку в TODO.md

**Застряла на вопросах Designer** → напиши "хватит вопросов, делай assumptions и двигайся дальше"
