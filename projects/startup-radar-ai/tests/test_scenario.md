# Test Scenarios — Startup Radar AI

## Сценарий 1 — Базовый запуск (Golden Path)

**Условия:** оба ключа установлены, интернет доступен
**Команда:** `python src/main.py`

**Ожидаемый результат:**
- Лог показывает 6 сгенерированных запросов
- Лог показывает ≥ 20 уникальных кандидатов
- Лог показывает ≥ 15 прошедших фильтр релевантности
- Файл `output/report_YYYY-MM-DD.md` создан
- Таблица содержит ≥ 15 строк
- Время выполнения < 5 минут

---

## Сценарий 2 — Отсутствует EXA_API_KEY

**Условия:** `.env` без строки `EXA_API_KEY`
**Ожидаемый результат:**
```
ERROR: EXA_API_KEY not set. Check your .env file.
```
Скрипт завершается без трейсбека.

---

## Сценарий 3 — Отсутствует ANTHROPIC_API_KEY

**Условия:** `.env` без строки `ANTHROPIC_API_KEY`
**Ожидаемый результат:**
```
ERROR: ANTHROPIC_API_KEY not set. Check your .env file.
```

---

## Сценарий 4 — Мало результатов

**Условия:** `DAYS_BACK = 7` в config.py (очень короткий период)
**Ожидаемый результат:**
- Скрипт находит < 5 кандидатов
- Выводит: `ERROR: Only N candidates found. Try increasing DAYS_BACK in config.py.`
- Отчёт не создаётся (не сохраняется пустой файл)

---

## Сценарий 5 — Качество данных (ручная проверка)

После базового запуска:
1. Открыть `output/report_YYYY-MM-DD.md`
2. Случайно выбрать 5 стартапов из таблицы
3. Проверить по URL каждого: действительно ли это AI-компания в нише аналитики стартапов?
4. Цель: ≥ 4 из 5 корректных

---

## Сценарий 6 — Повторный запуск (идемпотентность)

**Условия:** запустить дважды в один день
**Ожидаемый результат:**
- Второй запуск перезаписывает `output/report_YYYY-MM-DD.md`
- Нет ошибок, нет конфликтов
