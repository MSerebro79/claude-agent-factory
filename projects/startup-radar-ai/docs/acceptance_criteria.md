# Acceptance Criteria — Startup Radar AI

## Функциональные критерии

- [ ] `python src/main.py` запускается без ошибок при наличии .env с ключами
- [ ] Создаётся файл `output/report_YYYY-MM-DD.md`
- [ ] Таблица содержит ≥ 15 стартапов
- [ ] Каждая строка таблицы содержит: название, сайт, бизнес-модель, хотя бы 1 метрику или moat
- [ ] Все стартапы имеют минимум 1 источник (URL)
- [ ] Скрипт завершается за < 5 минут
- [ ] При отсутствии EXA_API_KEY или ANTHROPIC_API_KEY — понятное сообщение об ошибке, не трейсбек

## Качество данных

- [ ] ≥ 4 из 5 случайно проверенных стартапов действительно являются AI-компаниями
- [ ] ≥ 4 из 5 действительно работают в нише анализа стартапов/инвестиций
- [ ] ≥ 85% полей таблицы заполнены (не "N/A")
- [ ] Нет дублирующихся строк (одна компания = одна строка)

## Надёжность

- [ ] При временной ошибке Exa API — скрипт делает retry, не падает
- [ ] При временной ошибке Claude API — скрипт делает retry, не падает
- [ ] Если найдено < 5 кандидатов — понятное сообщение вместо пустого отчёта
- [ ] Лог-файл создаётся в `logs/run_YYYY-MM-DD.log`
