## 1. Problem [REQUIRED]

VC-аналитик тратит 3–5 часов на ручной поиск и сравнение AI-стартапов в нише "анализ успешности стартапов". Информация рассеяна по Product Hunt, TechCrunch, LinkedIn, X/Twitter. Ручная сводка непоследовательна: каждый раз разная глубина, нет единой структуры для сравнения. Существующие инструменты (CB Insights, Tracxn) платные, не заточены под конкретную нишу и не дают аналитику по moat.

## 2. User [REQUIRED]

VC-аналитик и независимый исследователь. Базовый Python. Работает в одиночку. Ценит скорость и структуру. Хочет получить готовую таблицу для принятия решений, а не сырые данные. Может перезапустить скрипт раз в месяц для обновления.

## 3. Use Cases [REQUIRED]

- **Основной:** запустить скрипт с параметрами (ниша, период, количество стартапов) → получить markdown-таблицу с 20 стартапами: бизнес-модель, ключевые метрики, moat
- **Дополнительный:** пересмотреть через месяц — запустить снова, сравнить с предыдущим отчётом
- **НЕ входит в scope:** мониторинг в реальном времени, алерты, CRM-интеграция, оценка стоимости стартапов

## 4. Inputs [REQUIRED]

| Параметр | Тип | Значение по умолчанию | Источник |
|---|---|---|---|
| `topic` | string | "AI startup analytics tools" | config.py или CLI-аргумент |
| `days_back` | int | 60 | config.py или CLI-аргумент |
| `target_count` | int | 20 | config.py или CLI-аргумент |
| `stage_filter` | string | "early-stage, seed, pre-seed" | config.py |

## 5. Outputs [REQUIRED]

**Файл:** `output/report_YYYY-MM-DD.md`

Содержимое:
1. Заголовок, дата, параметры запроса
2. Сводная таблица 20 стартапов:

| # | Название | Сайт | Стадия | Бизнес-модель | Ключевые метрики | Moat | Упоминания | Дата основания |
|---|---|---|---|---|---|---|---|---|

3. Источники (список URL)
4. Методология и уровень уверенности

## 6. Workflow [REQUIRED]

1. Загрузить параметры из `config.py` или CLI
2. Генерировать 6 поисковых запросов через LLM (разные углы: Product Hunt, funding, media, LinkedIn)
3. Выполнить поиск через Exa API с фильтром по дате (`days_back`)
4. Дедублировать результаты по домену
5. Отранжировать по частоте упоминаний (количество уникальных источников)
6. Взять топ-30 кандидатов
7. Для каждого из 30: запросить у LLM анализ (бизнес-модель, метрики, moat) на основе сниппетов из поиска
8. Отфильтровать нерелевантные (не AI, не early-stage, не аналитика стартапов)
9. Взять топ-20 по релевантности
10. Сгенерировать итоговую markdown-таблицу через LLM
11. Сохранить в `output/report_YYYY-MM-DD.md`

**Fallback на каждом шаге:**
- Шаг 3: если Exa недоступен → логировать ошибку, завершить с сообщением
- Шаг 7: если LLM возвращает неполный ответ → заполнить поле "N/A", продолжить
- Шаг 9: если найдено < 20 стартапов → включить в отчёт сколько есть, указать в заголовке

## 7. Agent Architecture [REQUIRED]

**Нужен ли агент?** Нет — достаточно скрипта с LLM-вызовами.

Архитектура: **одиночный Python-скрипт** с последовательными LLM-вызовами.

- Нет оркестратора
- Нет параллельных агентов
- LLM используется как инструмент (query generation + analysis), а не как агент с памятью

**Обоснование:** задача линейная, детерминированный порядок шагов, нет необходимости в планировании или инструментах с состоянием.

## 8. Tools & Stack [REQUIRED]

- [x] **Minimal** — Python + Claude API + Exa Search API

| Компонент | Инструмент |
|---|---|
| LLM | claude-sonnet-4-20250514 |
| Поиск | Exa Search API (семантический, с датой) |
| База | Нет (stateless, вывод в файл) |
| Автоматизация | Нет (ручной запуск) |
| Интерфейс | CLI (`python src/main.py`) |

## 9. Data Model [REQUIRED]

**Stateless — данные не сохраняются в базе.**

Промежуточные структуры в памяти:

- `SearchResult`: `{url, title, snippet, published_date, source}`
- `StartupCandidate`: `{name, url, mention_count, sources[]}`
- `StartupProfile`: `{name, url, stage, business_model, key_metrics, moat, confidence, sources[]}`

Выходной файл: `output/report_YYYY-MM-DD.md` — сохраняется локально.

## 10. LLM Responsibilities [REQUIRED]

- Генерация поисковых запросов (6 штук, разные углы)
- Оценка релевантности кандидата по сниппету (is it AI? early-stage? startup analytics?)
- Извлечение: бизнес-модель, ключевые метрики, moat из сниппетов
- Синтез финальной markdown-таблицы
- Оценка уровня уверенности по каждой строке

## 11. Code Responsibilities [REQUIRED]

- Загрузка конфига и CLI-аргументов
- Вызов Exa API + обработка ответа
- Дедупликация по домену
- Сортировка по `mention_count`
- Запись файла output
- Логирование (stdout + `logs/run_YYYY-MM-DD.log`)
- Обработка HTTP-ошибок и таймаутов

## 12. Human Review Points [REQUIRED]

- Просмотр итоговой таблицы перед использованием — LLM может ошибаться в метриках
- Проверка источников по 2–3 стартапам из топа — верификация данных

Всё остальное: **fully automated**.

## 13. Prompts Needed [REQUIRED]

- `query_generator`: принимает тему и параметры → выдаёт 6 поисковых запросов на английском
- `relevance_checker`: принимает сниппет → возвращает `{relevant: bool, reason: str}`
- `startup_profiler`: принимает название + сниппеты → выдаёт `{business_model, key_metrics, moat, confidence}`
- `table_synthesizer`: принимает список StartupProfile → форматирует финальную markdown-таблицу

## 14. Integrations

- **Exa Search API** — семантический веб-поиск с фильтром по дате. Ключ в `.env` как `EXA_API_KEY`.
- **Anthropic API** — Claude Sonnet. Ключ в `.env` как `ANTHROPIC_API_KEY`.

## 15. Risks [REQUIRED]

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| LLM галлюцинирует метрики (ARR, пользователи) | Высокая | Среднее | Указывать уровень уверенности и источник для каждого поля; пользователь проверяет топ-5 |
| Exa возвращает < 20 релевантных стартапов | Средняя | Среднее | Fallback: расширить `days_back` до 90, снизить строгость фильтра |
| Стартап не является early-stage (ошибка классификации) | Средняя | Низкое | Явный фильтр в `relevance_checker`; финальная проверка в таблице |
| Дублирование (одна компания под разными именами) | Низкая | Низкое | Дедупликация по домену + LLM-проверка на схожесть |
| Rate limit Exa или Anthropic API | Низкая | Низкое | Exponential backoff в коде |

## 16. Kill Decision [REQUIRED]

**Verdict:** СТРОИТЬ

Обоснование: задача повторяемая (раз в месяц), ручной вариант занимает 3–5 часов, автоматизация даёт 10–15-кратное ускорение при стоимости < $0.50 за запуск. Прямой Claude-запрос без Exa не даст свежих данных и источников.

| Exa Search | ~50 запросов | — | $0.00 (free tier: 1K req/month) |

|---|---|

## 19. MVP Scope [REQUIRED]

Входит в MVP:
- `src/main.py` — основной скрипт
- `src/config.py` — параметры
- `src/prompts.py` — все 4 промпта
- `.env.example` — шаблон ключей
- `output/` — директория для отчётов
- Markdown-таблица с 20 стартапами

НЕ входит в MVP:
- Telegram-уведомление по завершении
- Сравнение с предыдущим отчётом (diff)
- Веб-интерфейс
- Автозапуск по расписанию (cron)
- Scoring/ranking стартапов по инвестиционной привлекательности

## 20. Non-goals [REQUIRED]

- Не является инвестиционным советником
- Не заменяет due diligence
- Не работает с закрытыми базами данных (CB Insights, Crunchbase Pro)
- Не хранит данные о стартапах (нет CRM-функций)
- Не принимает решения — только структурирует публичную информацию

## 21. Guardrails [REQUIRED]

```
DO NOT:
- публиковать результаты без проверки источников
- использовать метрики LLM как достоверные факты без верификации
- интерпретировать таблицу как инвестиционную рекомендацию
- продолжать при отсутствии EXA_API_KEY или ANTHROPIC_API_KEY
- записывать данные в файл если найдено < 5 стартапов (аварийный выход с сообщением)
```

## 22. Evaluation Plan [REQUIRED]

| Метрика | Как измерять | Цель |
|---|---|---|
| Релевантность стартапов | ручная проверка 5 случайных из 20 | ≥ 4 из 5 действительно AI + early-stage + аналитика |
| Полнота данных | доля заполненных полей таблицы | ≥ 85% полей не "N/A" |
| Время выполнения | время от запуска до файла | < 5 минут |
| Стоимость запуска | API cost log | < $1.00 |
| Уникальность | нет дублей в таблице | 0 дублей |

## 23. Build Plan

- **V0:** этот blueprint + ручной поиск в Claude
- **V1:** `src/main.py` — автоматизированный скрипт (MVP)
- **V2:** + diff с предыдущим отчётом, + Telegram-уведомление
- **V3:** + scoring по инвестиционным критериям, + интеграция с Notion/Obsidian

## 24. Acceptance Criteria [REQUIRED]

- [ ] Скрипт запускается командой `python src/main.py` без ошибок
- [ ] Создаётся файл `output/report_YYYY-MM-DD.md`
- [ ] Таблица содержит ≥ 15 стартапов (допуск если Exa вернул меньше)
- [ ] Каждая строка содержит: название, сайт, бизнес-модель, хотя бы 1 метрику, moat
- [ ] Все стартапы с источниками (минимум 1 URL на стартап)
- [ ] Скрипт завершается за < 5 минут
- [ ] При отсутствии API-ключей — понятное сообщение об ошибке
