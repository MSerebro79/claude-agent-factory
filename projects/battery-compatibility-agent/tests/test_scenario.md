# Test Scenarios: Battery Compatibility Agent

## Запуск тестов

Запускай каждый сценарий вручную и сверяй результат с ожидаемым.

---

## Сценарий 1: Профессиональный инструмент (Makita)

```bash
python3 src/main.py "BL1850B"
```

**Ожидаемый результат:**
- Brand: Makita
- Series: 18V LXT упомянута
- Models: DDF484 или аналогичные
- Confidence: High
- Sources: ссылка на makitatools.com или amazon

**Проверка на галлюцинации:** открой первый source, убедись что упомянутые модели там есть.

---

## Сценарий 2: Потребительская электроника (Apple)

```bash
python3 src/main.py "iPhone 14 battery"
```

**Ожидаемый результат:**
- Brand: Apple
- Devices: iPhone 14, iPhone 14 Plus
- Нет устройств других брендов
- Confidence: High

---

## Сценарий 3: Обобщённый артикул

```bash
python3 src/main.py "18650"
```

**Ожидаемый результат:**
- Широкий список устройств (фонарики, ноутбуки, инструменты)
- Confidence: Medium (много брендов, нет однозначного производителя)

---

## Сценарий 4: Несуществующий артикул

```bash
python3 src/main.py "ZZZBAT99999"
```

**Ожидаемый результат:**
- Нет падения
- Confidence: Low
- Сообщение: не удалось найти данные
- Нет выдуманных устройств

---

## Сценарий 5: Exa API error (ручной тест)

Временно замени EXA_API_KEY на невалидный в `.env`, запусти любой запрос.

**Ожидаемый результат:**
- Понятное сообщение об ошибке
- Нет traceback в выводе пользователя
- Exit code != 0
